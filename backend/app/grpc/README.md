# gRPC слой

gRPC используется для вызовов между сервисами. REST остается основным интерфейсом для фронта.

## Что важно

- Контракт лежит в `backend/proto/task_service.proto`, stubs генерируются скриптом.
- Авторизация такая же, как в REST: Bearer-токен в metadata.
- gRPC-методы используют те же доменные сервисы, поэтому бизнес-логика не дублируется.

## Что можно добавить дальше

- Новые RPC без ломки текущих клиентов.
- Streaming-методы для push-сценариев.
- Отдельный gateway-слой, если потребителей станет больше.
