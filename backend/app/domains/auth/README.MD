### Аутентификация и авторизация

В этом слое логика отделена через порты (`ports.py`), чтобы не привязывать приложение к одному способу входа.

- Можно подключить внешний провайдер (Keycloak, LDAP, OAuth) без переписывания роутеров и сервисов.
- Локальная таблица `users` оставлена как компромисс для тестового задания: сервис запускается автономно, без какой-либо зависимости от сервиса юзеров условно.
- Если появится отдельный user-service, `UserStore` можно заменить на `RemoteUserStore`, а остальной код не менять.
