# Домен пользователей

Этот слой отвечает за прикладные операции с пользователями: создание, синхронизацию, смену/сброс пароля и выборку пользовательских задач.

## Ключевые решения

- Введен порт `UserStore`, чтобы бизнес-логика не зависела от конкретного хранилища.
- Текущая реализация использует локальную БД для автономности в рамках ТЗ.
- Поддержан `sync`-сценарий для интеграции с внешним источником пользователей.

## Точки роста на будущее

- Подключение `RemoteUserStore` (HTTP/gRPC) без изменения роутеров и сервисов.
- Расширение модели ролей (RBAC/ABAC) и групп без переписывания доменных use-case.
- Переход на event-driven синхронизацию пользователей вместо периодических вызовов.
