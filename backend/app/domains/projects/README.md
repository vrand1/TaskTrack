# Домен проектов

Проекты нужны как контекст для группировки задач и ограничения области работы команды.

## Ключевые решения

- Проект — отдельная сущность, а не просто поле в задаче: это упрощает права, фильтрацию и навигацию.
- Для проектов используется soft-delete с возможностью восстановления.
- Операции по проектам согласованы с задачами (ссылочная целостность и проверка ограничений).

## Точки роста

- Ввести отдельные роли на проект (owner/maintainer/member).
- Добавить метрики проекта (прогресс, скорость, доля reopen/просрочек).
- Расширить интеграцию с внешними системами планирования (Jira/YouTrack и др.).
